# Contributing to StayWithMe

First of all, thank you for taking the time to contribute to StayWithMe! 

StayWithMe is an open-source project focused on making smartphone learning more accessible, supportive, and less intimidating. Every contribution, whether it's documentation, design, research, bug fixes, or code, helps move the project closer to that goal.

# Ways You Can Contribute

There are many ways to contribute, including:

- Improving documentation
- Reporting bugs
- Suggesting new features
- Sharing research and accessibility insights
- Improving the user experience
- Writing or reviewing code
- Testing new features

No contribution is too small.

# Before You Start

Before working on a major change, please open an Issue to discuss your idea.

This helps avoid duplicated work and ensures proposed changes align with the project's goals and vision.

# Development Guidelines

When contributing, please aim to:

- Keep changes focused and easy to review.
- Write clear and meaningful commit messages.
- Update documentation when introducing new features.
- Follow the project's design principles.
- Prioritize accessibility and simplicity.

# Pull Requests

Before submitting a Pull Request, please make sure:

- Your changes have been tested where applicable.
- Documentation has been updated if necessary.
- Your commit history is clean.
- The Pull Request clearly explains what was changed and why.

# Community Values

StayWithMe is built on empathy, patience, and respect.

We welcome contributors from all backgrounds and experience levels.

Please be kind, constructive, and respectful in all interactions.

# Questions?

If you're unsure about anything, feel free to open an Issue and start a conversation.

Thank you for helping make StayWithMe a project that empowers people to learn technology with confidence.
